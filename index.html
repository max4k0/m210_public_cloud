Thema Cloud:

Cloud heisst für mich erstmal nur, dass es ein System ist, welches über das Internet abrufbar ist.
Ich brauche daher physisch nur Internet Zugriff und damit kann ich mit jedem Gerät die Daten oder die Anwendung abrufen.

Vorteil ist klar. Zugriff auf Daten von überall. Aber nur weil es Google Drive gibt, heisst es nicht, dass das das einzige ist, das die Cloud anbietet.

Die weiteren begriffe wären SAAS, PAAS, FAAS und noch ein viertes.
Hier sagt der erste buchstabe meist, was alles angeboten wird und was für dich übernommen wird.
PAAS ist platform abhängig. Du bekommst ein OS wie Linux bereitgestellt und dann kannst du deine Anwendung drauf ausführen.
Hier ist wichtig, dass auch libs wie Java oder python bereitgestellt werden müssen. Du selber hast nicht immer die möglichkeit diese zu installieren.

Offener ist da SAAS, welches komplett Systemfrei ist. Hier hast du die meisten Rechte, aber hier brauchst du auch das meiste Wissen, wie du es bedienen willst.

Hingegen bei FAAS wird dir alles gegeben. Selbst das Programm. Du lädst dir einfach alle tools aus dem Internet. Als Beispiel hätte ich hier einen Minecraft server.
Hier hast du kein OS oder sogar ein Filesystem. Du hast einfach nur zugriff auf die eine Anwendung mit samt der Einstellungen.

Schöne vorteile, aber die Nachteile sind leider auch gegeben. So ist die latenz immer ein Problem. Eine Anwendung auf dem Host auszuführen ist immer latenzfrei.
Hier sieht man z.b auch das Problem an cloudgaming. Die Leistung kann gegeben sein, aber die Latenz bleibt immer da. Egal wie gering sie ist.

Sonnst gibt es noch Serverkosten und Serverwartung, worum man sich kümmern muss. Wobei hier FAAS, PAAS etc zur hilfe kommen, sodass zumindest die Serverwartung gerne in andere Hände gelangt.
